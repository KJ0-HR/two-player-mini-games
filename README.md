# 刷双人小游戏

这是一个双人联机小游戏项目。支持本地测试，也可以部署到公网后让不同城市的玩家一起玩。

## 启动

双击 `start.bat`，浏览器会自动打开：

```text
http://localhost:5177
```

如果另一位玩家在同一局域网内，把页面里的邀请链接发给对方即可。若要让其他设备访问，需要把链接里的 `localhost` 换成运行电脑的局域网 IP。

## 不同城市玩家使用

不同城市不能直接使用 `localhost` 或局域网 IP，需要把本项目部署到公网服务器。

部署后，两位玩家都打开同一个公网网址，例如：

```text
https://你的项目地址.onrender.com
```

房主创建房间，把页面里的邀请链接发给对方即可。对方不需要 `start.bat`，也不需要项目文件。

### 推荐部署到 Render

1. 把本文件夹上传到 GitHub 仓库。
2. 打开 Render，选择 New Web Service。
3. 连接 GitHub 仓库。
4. Render 会读取 `render.yaml`，自动使用：
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Health Check: `/api/health`
5. 部署完成后，把 Render 给你的 `https://...` 网址发给玩家。

项目已支持 `DATA_DIR`，部署时账户文件会保存在持久化目录中。摄像头功能需要 HTTPS，Render 这类公网平台默认满足。

### 其他平台

如果使用 Railway、Fly.io、服务器或宝塔面板，只要能运行 Node.js 18+，启动命令使用：

```text
npm start
```

需要开放平台提供的公网端口。程序会自动读取平台注入的 `PORT`。

## 当前功能

- 登录或注册账户。
- 游戏ID唯一，不能重复注册。
- 每个账户由游戏ID和密码决定。
- 可选择本地图片作为头像，头像会压缩后保存到账户中。
- 账户信息和头像保存在本文件夹的 `accounts.json` 中。
- 房主选择 1 到 5 号游戏，然后一键创建房间。
- 系统自动生成 4 位数字房间号。
- 玩家二只需要输入房间号即可加入。
- 最多 2 位玩家进入同一房间。
- 显示玩家列表和准备状态。
- 游戏 1 已接入“数学计算能力”。
- 游戏 2 已接入“五子棋对决”。
- 双方都准备后可以开始游戏。
- 数学题双方同步显示，限时 1 分钟。
- 先答对的一方得 1 分；答错则对方得 1 分；超时无人得分并进入下一题。
- 五子棋采用 25x25 棋盘，棋子落在线的交点上，黑棋先手，横竖斜任意方向连成五子获胜。
- 五子棋棋盘左右显示双方头像。
- 五子棋支持聊天，文本会在对应头像旁显示为气泡并保留 6 秒。
- 五子棋刚落子的一方可以申请悔棋，由对方选择条件：打开摄像头 10 秒，或输入对方指定文字。
- 摄像头悔棋会把申请者摄像头画面同步给对方；文字悔棋成功时，输入内容会像聊天一样显示为气泡。

## 后续接游戏的位置

- 前端入口：`app.js` 里的 `startGame` 点击事件。
- 游戏画面容器：`index.html` 里的 `.stage` 区域。
- 联机状态接口：`server.mjs` 里的房间和玩家状态，可继续扩展游戏数据同步。
